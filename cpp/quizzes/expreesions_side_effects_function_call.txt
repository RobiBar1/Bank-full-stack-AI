defult order of __cdecl in:

stage1: caller make the field ready - 
* save registers(EDX, EAX, ECX): that stage happned only if she need to save there values after the call.
* push arguments: she push the arguments to the stack from last to first for the first argunemnt will be in the head of stack.
* make call command: this automate push the "return adress"(value of EIP register that point to next line of code) to the stack,
and give the controll to the calle function.

stage2: start of calle function(Prolog) - 
* save old EBP: save the EBP of caller function.
* make new EBP: make EBP value the val of this function.
* allocate memory: for local variables and for tmp storge, will do it by offsets of bytes need from ESP(sub esp, 20).
* save addition registers: if function gona use EDI, EBX, ESI she need to push them to stack for be able use them later.

stage3: after calle end run, the things she need to do before return(Epilog) -
* make return value: if its <= 4 bytes it will store in EAX register, else  will story in hidden pointer that the function get.
* restore registers: pop and resotre EDI, EBX, ESI if saved them in start of running.
* frame destroy: realese local varibles memory and return EBP to the caller function.
* return: pop the return adress from stack to EIP and return the controll to the place where call the function.

stage4: caller actions after return to him to clean the calle -
* clean argument of calle: will return the ESP to where was(if we offest him 20 bytes before, so now will return that 20 bytes).
* save the resault: save the return adress from EXA to appropriate variable.
* restore registers: if caller pushed the registers EDX, EAX, ECX in stage 1 so now he will pop them, and after that the
head of the stack will be where it was before call to calle function.
